常用的几种排序算法

冒泡排序（BubbleSort）

​ 基本思想：两个数比较大小，较大的数下沉，较小的数冒起来。

​ 过程：

  • 比较相邻的两个数据，如果第二个数小，就交换位置。
  • 从后向前两两比较，一直到比较最前两个数据。最终最小数被交换到起始的位置，这样第一个最小数的位置就排好了。
  • 继续重复上述过程，依次将第2.3…n-1个最小数排好位置。

HashSet

以HashSet为例说明：

1. 无序性：不等于随机性。存储的数据在底层数组中并非按照数组索引的顺序添加，而是根据数据的哈希值决定的。
   2. 不可重复性：保证添加的元素按照equals()判断时，不能返回true.即：相同的元素只能添加一个。

添加元素的过程：以HashSet为例：
我们向HashSet中添加元素a,首先调用元素a所在类的hashCode()方法，计算元素a的哈希值，此哈希值接着通过某种算法计算出在HashSet底层数组中的存放位置（即为：索引位置），判断 数组此位置上是否已经有元素：
如果此位置上没有其他元素，则元素a添加成功。 —>情况1
如果此位置上有其他元素b(或以链表形式存在的多个元素），则比较元素a与元素b的hash值：
如果hash值不相同，则元素a添加成功。—>情况2
如果hash值相同，进而需要调用元素a所在类的equals()方法：
equals()返回true,元素a添加失败
equals()返回false,则元素a添加成功。—>情况2

           对于添加成功的情况2和情况3而言：元素a 与已经存在指定索引位置上数据以链表的方式存储。
            jdk 7 :元素a放到数组中，指向原来的元素。
            jdk 8 :原来的元素在数组中，指向元素a
            总结：七上八下

HashSet底层：数组+链表的结构。

==与equals()的区别

一、==

​ ==是一个运算符，可以使用在基本数据类型和引用数据类型中

​ 如果比较的是基本数据类型变量 ：比较两个变量保存的数值是否相同（类型不一定要相同）

​ 如果比较的是引用数据类型：比较的是两个变量的地址值是否相同（两个变量所指向的是不是堆中的一个对象实体）

二、equals()

​ equals()是一个方法，只能被对象调用，所以它只能比较引用数据类型。

因为它是一个方法所以它只能比较引用数据类型

​ equals()是java.lang.Object类的方法 ，如果不重写的话是和==相同，比较的是引用数据类型的地址值

String、Date、File、包装类等 都重写了继承自Object的equals（）方法，重写后比较的不是引用地址是否相同，而是比较两个对象的 “实体内容“是否相同。

自己写的类，也需要重写equals（） 可以自动生成。

  3. 有没有可比性，是不是同一种类（包括子类）
  4. 将比的对象强转为完全相同的一个类（同一个类的值类型）

Java的核心机制——垃圾回收

  • java消除了c/c++中需要程序员自己回收垃圾的责任，它提供了一种系统级跟踪存储空间分配情况，在JVM空闲时，检查并释放哪些可被释放的存储空间。

  • 垃圾回收在Java程序运行过程中自动进行，程序员无法精确控制和干预

  • Java程序会出现内存泄漏和内存溢出的问题（异常）

JDK JRE JVM

  • JVM ：英文名称（Java Virtual Machine），就是我们耳熟能详的 Java 虚拟机。它只认识 xxx.class 这种类型的文件，它能够将 class 文件中的字节码指令进行识别并调用操作系统向上的 API 完成动作。所以说，jvm 是 Java 能够跨平台的核心
  • JDK ：英文名称（Java Development Kit），Java 开发工具包。jdk 是整个 Java 开发的核心，它集成了 jre 和一些好用的小工具。例如：javac.exe，java.exe，jar.exe 等。
  • JRE ：英文名称（Java Runtime Environment），我们叫它：Java 运行时环境。它主要包含两个部分，jvm 的标准实现和 Java 的一些基本类库。它相对于 jvm 来说，多出来的是一部分的 Java 类库。
  • 层层的嵌：JDK>JRE>JVM

​ 我现在已经大三快结束了，这一段时间很焦虑。乱七八糟的愁啊。每天晚上很晚才睡，一遍又一遍的刷着乱七八糟的信息，加上被贩卖焦虑的文章所裹挟，而后一天又一天的失眠。

​ 晚上睡不着，白天不想睡，又没有精力去学习、去运动，进一步又导致我的焦虑。

​ 使我想要改变的契机是偶然在5.20看到了韩寒的《1988：我想和这个世界谈谈》中的一段话：

这样的旅行在我年少时曾经幻想过无数次，夜晚的国道里，我带着自己梦寐以求的女子，开着自己梦寐以求的车，去往未知旅程的终点。 未知的旅程怎么又会有终点 。 旅途上没有疲劳和困意，我们聊着电影和音乐，穿越群山和丛林，最终停在一泓无人的湖水旁边。有一个没有任何经济头脑的人开的酒店 干净、便宜。

​ ——1988：我想和这个世界谈谈

​ 我看完了这本书，这本书和我对韩寒一直以来的印象完美契合。故事也是一个经典的韩寒故事，但在这个节点却给我带来了前所未有的怅然若失。

​ 曾几何时，在我还未长大；在我还假装愤世嫉俗来显的自己高级；在我还对爱情，对长大后的世界，对以后的人生抱有最纯真而美好的想象时，我还是一个坚信未来是光明的，世界是充满希望的，我会有属于自己的光明而又美好的选择的 男孩。可现在我却是个表面上的言语上的积极乐观，一旦我对自己的各种念头稍加剖析便有喷涌而出的消极气味。满是腐臭的猥琐青年。用欲望填补空虚，用多巴胺抚慰身体的疲倦，用熬夜游戏与对自己的厌倦来使自己入眠 也许这是长大的必经的路，也或者只是我对自己脑补的矫情吧。虽然会矫情和烂俗的让自己起鸡皮疙瘩，我还是想说，我还是一个乐观的男孩。

​ 我现在开始想要搭建属于自己的博客，不再只是发些只言片语的说说，或者展现活给别人看的自己。

​ 我会这里会发些自己不时的矫情想法，来让未来的自己尴尬到脸酸。

​ 会在这里纪念一些逝去的东西，避免它被忘记以至于以后甚至怀疑其存在。

​ 会在这里写一些学习的东西，填补空虚。

呐 就这样吧。
